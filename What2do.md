# 개인 기록 웹사이트 – 최종 요구사항 명세서

## 1. 목적

이 웹사이트는 장기적으로 개인의 기록과 지식을 축적하기 위한
개인 기록 및 지식 아카이브 시스템이다.

이 웹사이트의 목적은 다음과 같다:
- 개인 학습 내용 정리
- 계획 및 회고 기록
- 일상 및 취미 기록
- 성취 사항 정리

이 웹사이트는 다음과 같은 특성을 가진다:
- 정적 웹사이트 (백엔드 없음, 데이터베이스 없음)
- HTML, CSS, JavaScript로만 구성
- 정적 호스팅 환경(예: Cloudflare Pages)에 배포
- 장기간 사용하며 점진적으로 확장하는 것을 전제로 설계

---

## 2. 최상위 섹션 구성

웹사이트는 다음과 같은 최상위 섹션으로 구성된다.

- Home
- Study
- Plan
- Diary
- Hobby
- Accomplishment

각 섹션은 하나 이상의 HTML 페이지로 구현한다.

---

## 2.1 Home 섹션

### 2.1.1 목적
Home 섹션은 웹사이트의 첫 화면으로, 웹사이트 소개와 개인 정보를 제공한다.

### 2.1.2 구성 요소
- 웹사이트 목적 및 설명
- 개인 소개 섹션
  - 왼쪽: 개인 정보
    - 이름 (Name)
    - 이메일 (Email)
    - 소속/학교 (Affiliation/School)
    - 전공 (Major)
    - 관심 분야 (선택 사항)
    - 소셜 링크 (GitHub, LinkedIn 등, 선택 사항)
  - 오른쪽: 프로필 사진 영역
- 섹션 목록 및 설명

### 2.1.3 레이아웃
- 개인 소개 섹션은 2열 레이아웃 (정보 + 사진)
- 반응형: 모바일에서는 세로 배치

---

## 3. Study 섹션

### 3.1 목적
Study 섹션은 분야별 학습 내용을 기록하고 정리하기 위한 공간이다.
시간이 지남에 따라 내용이 누적되는 구조를 가진다.

### 3.2 구조

Study 섹션은 다음과 같은 도메인으로 나뉜다.

- CS
- Math
- Web

#### CS 하위 분류
CS는 다음과 같은 하위 주제로 나뉜다.
- Algorithms
- OS
- LLM
- GPU
- NPU

각 하위 주제는 **개별 HTML 페이지**로 구현한다.

### 3.3 학습 내용 구조
- 도메인별 인덱스 페이지는 학습 항목 목록을 표시
- 각 학습 항목은 개별 HTML 파일로 분리
- 파일명은 날짜 기반 (예: 2025-12-27-web-build.html)
- 목록에서 제목을 클릭하면 해당 학습 내용 페이지로 이동

### 3.4 설계 원칙
- 주제 기반 도메인 구조
- 각 학습 내용은 개별 페이지로 관리
- 도메인 간 이동이 명확한 네비게이션 제공

---

## 4. Plan 섹션

### 4.1 목적
Plan 섹션은 개인의 목표와 계획을 기록하기 위한 공간이다.

### 4.2 구조
Plan 섹션은 시간 단위로 나뉜다.
- 주간 계획 (Weekly)
- 월간 계획 (Monthly)

각 계획은 개별 HTML 페이지로 구현한다.

### 4.3 상호작용
- 체크리스트 사용 가능
- JavaScript를 이용한 완료 표시 허용
- 완료 상태는 브라우저 로컬 상태로 저장 가능

서버 측 저장은 필요하지 않다.

---

## 5. Diary 섹션

### 5.1 목적
Diary 섹션은 주간 단위의 회고 및 일기 기록을 위한 공간이다.

### 5.2 구조
- 일기는 **주 1회 작성**
- 각 일기는 특정 주(week)에 대응
- 연도 단위로 정리

### 5.3 설계 원칙
- 시간 기반 구조
- 각 주차는 개별 HTML 파일로 구현
- 작성 이후 수정은 거의 발생하지 않음

---

## 6. Hobby 섹션

### 6.1 목적
Hobby 섹션은 일상 활동과 취미를 기록하기 위한 공간이다.

### 6.2 카테고리
- Exercise
- Cooking
- Singing

각 카테고리는 개별 HTML 페이지로 구현한다.

### 6.3 콘텐츠 성격
- 자유 형식의 글
- 이미지 포함 가능
- 엄격한 구조 요구 없음

---

## 7. Accomplishment 섹션

### 7.1 목적
Accomplishment 섹션은 그동안 이룬 성과를 정리하는 공간이다.

### 7.2 구조
- 연도별로 정리
- 각 연도는 개별 HTML 페이지로 구현

### 7.3 콘텐츠 예시
- 완료한 프로젝트
- 학업·기술적 성취
- 개인적인 성과

이 섹션은 다른 섹션보다 더 **정제되고 요약된 형태**를 유지한다.

---

## 8. 기술적 제약 조건

### 8.1 사용 가능 기술
- HTML
- CSS
- JavaScript (브라우저 실행, 최소 사용)

### 8.2 사용 금지 기술
- 백엔드 서버
- 데이터베이스
- 인증/로그인 시스템
- 초기 구현 단계에서의 프레임워크 (React, Vue 등)

---

## 9. 설계 철학

- 시각적 완성도보다 **구조와 내용**을 우선
- 장기 유지보수를 고려한 단순한 설계
- 파일 및 디렉터리는 의미가 명확해야 함
- 콘텐츠가 많아져도 사용 가능해야 함

이 문서는 **현재 기준의 최종 요구사항**을 정의한다.
